Chapter I — The Brief

Claude Design is a portable Claude skill — a SKILL.md plus a handful of on-demand references — that turns the model into a senior designer the moment your request sounds like design. Not a bigger model, not a new tool. A different set of defaults.

Once loaded, the skill rewires Claude's instincts. Before assuming anything about a named product it runs a search. Before reaching for colour and font it gathers logos, product shots, and UI screenshots. Before committing to a style it proposes three directions and waits.

The output is not prettier, exactly. It is more committed. Every choice cites a system; every variation holds to that system's rules.

"The default failure mode of AI design is not ugliness — it is sameness." — Rev 01 · 2026

Chapter II · Operating principles
I.

Verify facts before assuming.

When a specific product, version, or event is named, WebSearch comes before design. A 10-second search beats 1–2 hours of rework on a wrong premise. Non-negotiable.

II.

The Core Asset Protocol.

Ask → search → download → verify → freeze. Logo, product shots, UI screenshots are first-class citizens; colors and fonts are auxiliary. Skipping the first three is how agents produce generic tech design.

III.

The Direction Advisor.

When the brief is too vague to execute, propose three directions from ten philosophies across five schools. Build a small preview. Ask the user to pick — then commit fully.
