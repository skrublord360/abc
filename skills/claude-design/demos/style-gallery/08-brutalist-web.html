02. Three rules, in order of priority

[01] Priority Zero — verify facts before designing

When the brief names a specific product, company, version, or recent event, the first action is not a clarifying question and not design exploration. The first action is WebSearch. A 10-second search beats 1–2 hours of rework on a wrong premise.

WebSearch → read 1-3 authoritative results → write findings to product-facts.md → only then design.

[02] The Core Asset Protocol

For any task that names a specific brand, follow the five-step hard flow. Logo, product shots, and UI screenshots are first-class citizens. Colors and fonts are auxiliary.

StepActionOutput
1Askfull six-asset checklist requested from user
2Searchofficial channels probed — brand site, press kit, docs
3Downloadthree-path fallbacks · 5-10-2-8 quality rule
4Verifyeach asset real, hi-res, current, uncontaminated
5Freezebrand-spec.md written with paths, colors, fonts, gaps

[03] The Design Direction Advisor

When the brief is too vague to execute, do not improvise. Switch into Advisor mode: propose three directions drawn from different schools among the library of ten. Build a small preview. Ask the user to pick. Then commit fully.

The ten: 01 Swiss Editorial · 02 Bauhaus Geometric · 03 Kenya Hara · 04 Dieter Rams · 05 Magazine Editorial · 06 Zine/Riso · 07 Field.io · 08 Brutalist Web · 09 Sagmeister · 10 Y2K Futurist

Works across 40+ coding agents (Claude Code, Codex, Cursor, OpenCode, Gemini CLI, Cline, ...). Four install paths, pick one:
